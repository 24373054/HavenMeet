## 📈 二、调研发现

### 2.1 课程基本信息

**课程名称**: 软件工程基础  
**授课单位**: 北航计算机学院/软件学院  
**课程定位**: 专业基础课程，强调工程化方法和团队协作

**核心教学内容**:
- UML 建模 (用例图、类图、时序图)
- 需求分析与设计
- 编码与实现 (Django/Vue/Java)
- 测试与维护
- 项目管理与团队协作

### 2.2 小组项目特点

#### 团队规模
- **北航**: 5-6 人一组 (较大规模)
- **对比**: 其他学校通常 2 人一组，甚至个人完成
- **意义**: 强调大型团队协作能力

#### 项目类型 (按频率排序)

**🥇 传统管理系统类 (最常见)**
1. 图书管理系统
2. 项目管理系统
3. 超市管理系统
4. 学生选课系统
5. 网上招聘系统

**特点**:
- 功能明确，易于实现
- 数据结构清晰
- 适合作为教学案例
- 但创新性较低

**🥈 创新技术类 (逐渐增多)**
1. 智能工业软件
2. 大数据分析平台
3. AI 辅助开发工具
4. 开源协作平台

**特点**:
- 结合前沿技术
- 有实际应用价值
- 展示技术实力
- 但实现难度较高

### 2.3 评分标准 (推测)

根据课程要求分析，评分可能包含：

| 维度 | 权重 | 说明 |
|------|------|------|
| 功能完整性 | 40% | 是否实现所有需求 |
| 代码质量 | 20% | 规范、可维护性 |
| 团队协作 | 15% | 分工、沟通、版本管理 |
| 文档质量 | 15% | 需求、设计、测试文档 |
| 创新性 | 10% | 是否有亮点 |

### 2.4 技术栈要求

**前端**:
- Vue.js (课程明确提到)
- 可能涉及：Element Plus、Vuetify 等 UI 框架

**后端**:
- Django (课程明确提到)
- Java (面向对象程序设计)
- RESTful API 设计

**数据库**:
- 关系型数据库 (MySQL/PostgreSQL)
- 可能涉及：Redis 缓存

**工具**:
- Git (版本控制)
- UML 工具 (StarUML/PlantUML)
- 项目管理工具 (Jira/Trello)

## 💡 三、创新项目推荐

基于调研结果和当前技术趋势，为 5-6 人团队推荐以下项目：

### 🏆 推荐项目 1: AI 辅助代码审查平台

**创新指数**: ⭐⭐⭐⭐⭐  
**实现难度**: ⭐⭐⭐  
**展示效果**: ⭐⭐⭐⭐⭐

#### 项目概述
开发一个集成 AI 代码审查能力的协作平台，帮助团队提高代码质量。

#### 核心功能

**1. 代码管理**
- Git 仓库集成
- 代码提交历史
- 分支管理

**2. AI 代码审查**
- 自动检测代码质量问题
- 生成改进建议
- 代码规范检查
- 安全漏洞扫描

**3. 团队协作**
- 多人协作审查
- 评论和讨论
- 问题跟踪
- 任务分配

**4. 数据可视化**
- 代码质量趋势图
- 团队贡献统计
- 问题分布分析
- 审查效率指标

#### 技术栈

```yaml
前端：
  - Vue3 + TypeScript
  - Element Plus
  - ECharts (可视化)
  - Monaco Editor (代码编辑)

后端：
  - Django + Django REST Framework
  - Celery (异步任务)
  - Redis (消息队列)

AI 集成:
  - CodeBERT / CodeT5
  - 或调用开源 API

数据库:
  - PostgreSQL
  - Redis (缓存)

部署:
  - Docker
  - Nginx
  - CI/CD (GitHub Actions)
```

#### 团队分工 (6 人)

| 角色 | 人数 | 职责 |
|------|------|------|
| 项目经理 | 1 | 需求分析、项目规划、文档 |
| 后端开发 | 2 | API 设计、数据库、AI 集成 |
| 前端开发 | 2 | UI 设计、页面开发、交互 |
| DevOps | 1 | 部署、CI/CD、测试 |

#### 开发计划 (10 周)

**Week 1-2**: 需求分析 + 系统设计
- 撰写需求规格说明书
- 绘制 UML 图
- 数据库设计
- 技术选型确认

**Week 3-5**: 核心功能开发
- 后端 API 开发
- 前端页面开发
- Git 集成
- 基础审查功能

**Week 6-7**: AI 集成 + 优化
- 集成 AI 代码审查
- 性能优化
- 用户体验优化

**Week 8**: 测试 + Bug 修复
- 单元测试
- 集成测试
- Bug 修复

**Week 9-10**: 文档 + 演示
- 编写完整文档
- 准备演示视频
- 项目总结

#### 创新点

1. **AI 赋能**: 结合大语言模型进行智能代码审查
2. **实时协作**: 多人同时审查，实时同步
3. **数据驱动**: 基于历史数据给出改进建议
4. **可视化**: 直观展示代码质量趋势

#### 预期成果

- ✅ 完整的 Web 应用
- ✅ AI 代码审查功能
- ✅ 团队协作平台
- ✅ 数据可视化大屏
- ✅ 完整的项目文档

### 7.1 调研结论

1. 强调团队协作 (5-6 人)
2. 注重工程化方法

4. 鼓励技术创新
